Authentication

This blog post is about authentication and authorization.

Authentication is the act or process of authenticating to verify or establish the identity of something, often a person. Authenticaion provides assertion the the person or computer is who they claim to be. A low tech example of authentication is your driver's license, which shows that the bearer of the I.D. is the same as information attached. Verification the person is who they claim can be accomplished by confirming the physical characteristics deatailed on the card match, thereby using attribute comparison to authenticate.

Some common methods of authentication are something the user knows (like a password, passphrase, PIN, or security questions), something the user has (I.D. card, hardware security token, or software security token), or something the user is (like fingerprint, retinal scan, gait analysis, or other biometric identifier). For modern authentication, multiple factors are considered more secure than single factor. For example, authentication using a password (something the user knows) in combination with a fingerprint scan (something the user is) is stronger than using a password alone. Using two authentication methods from the same category, such as password and security questions, is not considered multi-factor.

Given the ubiquitous nature of cell phones, it seems logical that these devices have be co-opted for use in authentication. Mobile devices can authenticate using a built-in hardware token when supported by the manufacturer, or they can use a software token using an authentication app. In the case of the software token, the authenticator app provides users with a OTP (also known as One Time Password) that users enter in a challenge / response prompt during authentication. These are often time-sesitive (referred to as Time-based One Time Password), so the user must enter within a set amount of time.

Software also uses authentication to verify the user is authorized. Software authenticaion can take the form of a product key, where a user enters an alphanumeric code to prove they're authorized for use. Another mechanism for authentication used by software is a hardware key that must be connected to the computer running the software. Connecting the physical security token to the device verifies the user is authorized to utilize the product.

Authentication is distinct from authorization because authentication only verifies an identity, whereas authorization says whether or not a user is authorized to access a resource. A user may authenticate to confirm they are who they claim but can still not be allowed access to a resource because they're not authorized. For example, a miltiary officer may scan an ID card to confirm their identity, but still not be allowed access to documents because they don't have the appropriate clearance. Used together, authentication and authorization combine for Access Control, which restricts access to a resource.

Mutual authentication is used when the server and client both authenticate to each other. Single Sign On (also known as SSO) is used to authenticate a user to multiple applications using a single set of credentials. Authorization techniques include RBAC (Role Based Access Control), SAML (Security Assertion Markup Language, OpenID, and OAuth.
