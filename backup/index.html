Backup and Recovery

This blog post is about backup and recovery.

Backup and recovery are critical business processes that maintain continuity and data integrity. Restoration from backup provides a recovery option in the event of a disaster. Creating backup of data involves storing duplicates of data in a secure location. The backup data represents a snapshot in time when the backup is created, and restoration from that backup represents a "Restore Point", that is the point in time of the last backup. Recovery of lost or damaged files is only possible from data that has been backed up, and only from the most recent copy of backup data.

A backup must be created before restoration from recovery is required. Backups must be stored in a secure location, preferably offsite from the business operations center. This ensures the backup is not compromised in the case that the primary operations center is destroyed. The recovery process must be validated to confirm it works as expected before it is needed to ensure faster recovery and sustainability of critical business processes. Restoration of data may be to the original location or an alternate location.

A generally accepted backup strategy is the "3-2-1" approach. It means having at least three backup copied of data: two local copies using different media (i.e. using different device, for exmaple a tape backup and server storage backup) and one remote copy (stored offsite from the primary business location). With the prevelance of ransomware attacks, some administators are expanding on the 3-2-1 method with "3-2-1-1-0". where the last "1" represents an offline copy of data (no accessible and therefore less vulnerable), and the "zero" represents ensuring zero errors in all recovery solutions.

There are different types of backups: full backups, incremental backups, differential backups, and mirror backups. Full backups completely duplicate all data included in the backup, which can take a long time to complete and require a lot of storage space. For data that doesn't change often, full backups often include the same data in sequential backups. Incremental backups start with a full copy of the data, but then only changes in the saved data that occurred since the last backup (full or incremental) are backed up; data captured after the last full backup are incrementally added to the backup. Incremental backups have the advantage of reduce time for backup and reduced storage space required, allowing companies to backup data more often. Incremental backups are generally faster to complete, but can result in increased recovery time because multiple incremental backups may need to be combined to restore data.

Differential backups capture all new and changed data since the last full backup, and a new full backup is taken incrementally (often once per week) to ensure data integrity. Differential backups generally take longer than incremental backups to create, but can be faster to restore from because only the latest differential backup and a copy of the full backup are required for restoration. Mirror backups are an identical copy of backup data. The initial mirrored copy of data is slow, but subsequent backups are faster because they only included the latest version of files. Drawbacks to mirror backups include accidental file deletion (deleting a file also removes it from the next mirrored backup copy), as well as not being able to track different version of files.

Some examples of incidents that could trigger the recovery process are natural disaster, ransomware, accidental deletion, or sabotage.
