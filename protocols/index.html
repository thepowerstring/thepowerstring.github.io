Protocols

This blog post is about secure protocols.

To protect the privacy and integrity of data, impement secure protols. These protocols can be categorized as routing protocols, mail transfer protocols, remote communication protocols, network security protocols, and many more.

Secure protocols are most often secure implementations of existing legacy protocols. Some examples include IPSec (Secure implementation of IP protocol), HTTPS (secure implementation of HTTP), SCP (or Secure Copy) based on SSH (Secure SHell), SFTP (Secure File Transfer Protocol), SNMP version3, and SMB version 3. SSH protocol can used as a tunneling method to secure other protocols, or as a standalone utility for connecting to remote hosts. SSH replaces the less secure Telnet, which does not encrypt transmitted data. Other implementations of secure protocols, like IMAPS, utilize SSL / TLS to facilitate secure communitcations. Kerberos is an example of a secure network protocol used to provide strong authentication between server and client.

The SSH protocol is the standard for secure protocols. As previously mentioned, it can be utilized as a wrapper to secure otherwise insecure protocols. The keys used for authentication in SSH are called 'SSH keys'. Key-based authentication does not require interactive login, so it can be utilized to automate business process. Once an SSH session between a client and server is established, data transmitted over the connection is secured through encapsulation inside the SSH tunnel. SSH supports a wide variety of encryption algorithms, including AES. In a business setting, it's important to implement policies that invalidate SSH keys when access is revoked.

PGP stands for Pretty Good Privacy, which is an encryption prgogram that uses cryptogrophy to secure communication. PGP is most commonly used to encrypt e-mails, for digital signatures, and for file encryption. To encypt an e-mail message using PGP, the sender uses the recipients public key and the PGP program to convert data from unencrypted plaintext to encrypted cyphertext, sends the message, and then the sender decrypts the message using their private key. An additional, optional step is to create a session key for each conversation, where only the session key is encrypted and sent to the recipient for decryption and they decryot the session key and then the message. Implementation of PGP encryption can be done using a program that supports it, like ProtonMail, or by using an extension, like Thunderbird, with your existing mail program.

PEM, or Privacy-Enhanced Mail, is another protocol used for email security. PEM protocol is capable of performing encryption, nonrepudiation, and message integrity. This protocol has been less widely adopted then PGP.

SET, also known as Secure Electronic Transaction, is a method for assuring security and integrity of electronic transactions made using credit cards. The SET protocol is not a payment system, but instead is the protocol for transmitting data over the Internet.

Whenever possible, secure protocols should be implemented to replace older, less secure protocols. Additional configuration is usually required to implement secure protocols, but they have the advantage of protecting data in transit when compared to legacy protocols that transmit data unencrypted, or "in the clear" as clear text.
