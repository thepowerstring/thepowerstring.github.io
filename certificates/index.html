Certificates

This blog post is about certificates.

A digital certificate is an electronic password in the form of a file, used for proof of authenticity in secure communications. In assymetric encryption, also known as public key encryption, the sender encrypts the data using the public key, and the recipient decrypts the message received using the private key. The two keys (public and private) are said to be "paired", meaning data encrypted with the public key can only be decrypted using the private key. Authentication of a public certificate is performed using cryptographic algorithms and the PKI (also known as Public Key Infrastructure), which is a network of publicly trusted Certificate Authorties (or CA for short).

Digital certificates can be used for authenticating to a the network, like the company wireless LAN. Digital certificates can also be used as a signature to affirm the authenticity of the sender, as in e-mail messages and software updates. Another common application of digital certificates is TLS (Transport Layer Security) or SSL (also known as Secure Socket Layer), used for confirmation of a website's secure connection to a web browser. For a website, a client obtains that site's public key from the SSL certificate to initiate secure communications, and the website maintains the private key as a secure secret.

TLS evolved from the SSL encryption protocol that preceded it, so the two terms are often used interchangeably. TLS is an encrypoion and authentication protocol that uses a process called the "TLS Handshake". During a TLS handshake, the two sides of a commincation exchange messages to acknowlege and then verify the other party. During this time, they also establish the cryptographic algorithms to be used and then agree on session keys. This technology is foundational to the operations of HTTPS, also known as HyperText Transfer Protocol Secure (the secure version of HTTP). TLS handshakes occur after any TCP connection has been opened, including API calls and DNS over HTTPS queries.

Information (such as full name, company, department, IP address, serial number, etc.) about the certificate's owner are normally attached as part of the certificate. Digital certificates also contain a copy of the public key of the certificate holder, which is used for decyphering data encypted with the private key. A public key certificate is issued by an established vendor called a Certificate Authority (or CA) that has a high level of trust on the Internet. When a client computer is presented with a certificate from a server, it runs certification path validation to ensure the subject of the certificate matches the host name. Certificates can be domain validated, in the case of websites, or organization validated, used for small and medium sized businesses.

A self-signed certificates is a certificate that isn't signed by a publicly trusted CA. Anyone has the ability to generate a new self-signed certifcate on their local machine. Self-signed certificates are generated, issed, and signed by the entity issueing the certificate. Another type of certificate is the code signing certificate

Symmetric encryption uses the same key to encrypt and decrypt the data, so both communicating parties must know what the key is. For this method of encryption to be effective, the secret key must not be able to be stolen or guessed.
