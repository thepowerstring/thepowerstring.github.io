VLANs

This blog post is about VLANs.

VLANs, or Virtual Local Area Networks, as the name implies, are synthetic representations of a physical LAN that is common to any local group of computers of the same subnet. VLANs allow logical segmentation of a broadcast domain from other comptuers on the same LAN. Virtual Local Area Networks are defined and supported under the 802.1q specification, sometimes referred to as Dot1q. This standard also includes provisios for Quality of Service (also known as QoS) prioritization.

VLANs work by taggin network frames at Layer 2 (also known as Data Link layer) of the OSI model, creating a virtual network that operates transparently as a standard LAN. When VLANs are used, any frames that are not tagged with a specific VLAN are assumed to be in the "default" VLAN, which often defaults to VLAN number one but can be chaned to a nonstandard value. VLANs can traverse multiple switches, as long as all network devices involved support VLAN tagging, which allows for connecting devices on the smae VLAN even if they are not connected to the same physical switch. The ability to segment networks in software also allows for traffic isolation without physically changing the physical wired configuration to use different switch ports.

VLANs are defined by configuring network devices, which can include switches, routers, and firewalls. One common example of VLANs is create a DMZ (also known as Demitilirized zone) for allowing traffic onto the network to access a public resource (like a web server). Another use of VLANs could be to segment traffic for like devices on their own VLAN (like VoIP phones on one VLAN, printers on another VLAN). By segmenting traffic into different logical networks, network performance can be improved because the amount of overall crosstalk between devices is reduced. Isolating broadcast domains reduces the number of devices on each domain, thereby reducing the broadcast traffic to all devices.

VLAN Access Control Lists, or VCALs, provide access control for packets that are bridged or routed to a VLAN. This can be helpful for network monintoring, in the case of Data Loss Prevention (DLP) or Network Intrusion Prevension (NIPs) systems. VLANs can be configured statically assigned based on a devices connected switchport, or dynamically using predefined characteristics of the devices (such as MAC address of network account namnes). VLAN segmentation can improve the overall security posture for a given LAN, but should not be solely relied upon as a security mechanism becuase it's possible for malicious actors to "hop" or traverse VLANs.

VLANs are isolated into separate broadcast doamins by default, but can be made to communiate with each other. Inter-VLAN routing can be used to allow different VLANs to talk to each other. In legacy IVR, multiple router interfaces are used to connect to a switch port that assigned to the respective VLAN. The so-called "Router-on-a-stick" method of inter-VLAN communciation uses a single router interface, and the router's interface is devided into subnets acting as a gatweay to the differnt VLANs.

SVIs, also known as Switched Virtual Intergaces, is usually supported on Layer 3 devices in the form of a virtual routed interface that connects VLANs. SVIs are usually much faster than the "Router-on-a-stick" model of inter-VLAN communication because switching and routing is handled at a hardware level. Latency is also reduced, because traffic does not need to leave the switch.
